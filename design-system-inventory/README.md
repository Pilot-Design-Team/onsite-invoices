# Design System Inventory

This directory contains a comprehensive inventory of the Pilot design system, extracted from the production codebase.

## Contents

### Documentation Files

1. **COMPONENTS_INDEX.md** - Complete catalog of all Vue components
   - Component categories (base, pilot, containers, forms, tables, etc.)
   - Component props, slots, and usage notes
   - Storybook coverage indicators

2. **TOKENS_INDEX.md** - Design tokens reference
   - Color tokens (base, semantic, status colors)
   - Typography tokens (font sizes, line heights)
   - Spacing tokens (2px to 240px scale)
   - Border radius, shadows, z-index, layout tokens
   - Token system maturity assessment

3. **FONTS.md** - Font system documentation
   - Font family details
   - Font file locations
   - Font loading configuration

4. **PATTERNS_INDEX.md** - UI patterns catalog
   - Form patterns (standard forms, wizards, inline editing)
   - Table patterns (data tables, filtering, transaction tables)
   - Modal patterns (dialogs, drawers, speedbumps)
   - Navigation patterns (primary nav, tabs, breadcrumbs)
   - Empty states, loading states, notifications
   - Pattern duplication issues

5. **LAYOUTS_INDEX.md** - Layout system documentation
   - PageWrapper and layout routing
   - Grid system and column layouts
   - Responsive breakpoints
   - Layout components (admin, portal, tax partner, logged out)
   - Container layouts (modal, drawer)

6. **OWNERSHIP_GAPS.md** - Gaps and recommendations
   - Legacy/deprecated components
   - Duplicated patterns
   - Missing Storybook coverage
   - Missing design tokens
   - Inconsistent naming
   - Documentation gaps

7. **STORYBOOK_COVERAGE.md** - Storybook analysis
   - Story-to-component mapping
   - Coverage gaps
   - Story organization and conventions
   - Recommendations for improvement

### Data Files

- **inventory.json** - Machine-readable inventory (generated by script)
- **scripts/extract_inventory.mjs** - Extraction script

## Token System Maturity

**Status:** Partial

### ✅ Strengths
- Comprehensive color system with semantic naming
- Consistent spacing scale (2px to 240px)
- Typography scale defined
- Z-index system organized
- Colors exported to JavaScript

### ⚠️ Gaps
- No CSS custom properties (primarily SCSS variables)
- No token JSON/YAML format
- Limited motion tokens (only one duration)
- Breakpoints not tokenized
- No design tool export

### Recommendations
1. Migrate to CSS custom properties for runtime theming
2. Create token JSON/YAML files (style-dictionary format)
3. Expand motion token system
4. Tokenize breakpoints
5. Generate token documentation

## Usage

### Viewing the Inventory

Read the markdown files in this directory for detailed information about each aspect of the design system.

### Running the Extraction Script

```bash
cd design-system-inventory/scripts
node extract_inventory.mjs
```

This will generate `inventory.json` with machine-readable component and story data.

### Updating the Inventory

The inventory should be regenerated when:
- New components are added
- Components are deprecated
- Storybook stories are added/removed
- Design tokens are updated
